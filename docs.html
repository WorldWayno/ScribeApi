Scribe Online API

The Scribe Online REST API enables you to manage your Solutions. Before you write any code, you can use your Scribe Online credentials and organization ID to use the API with our interactive API reference.

Authentication

Scribe Online API requests must be authenticated using HTTP Basic Authentication. To authenticate, add a header parameter:

  • The key is Authorization
  • The value is
  • The {email}:{password} section is base 64-encoded.

See documentation for your client HTTP library (such as jQuery) for information about adding a header parameter for HTTP Basic Authentication.

Versions

The current version of the API is v2. Specify the version at the base of the API URL path:

https://api.scribesoft.com/v1/...

Organizations

API requests are made in the context of an Organization, which is specified after the API version in the URL:

https://api.scribesoft.com/v1/orgs/{orgId}/...

The Organization ID is a number is found in Scribe Online, on the Edit Organization screen. If you have not joined an Organization, one is created for you when you first log in to Scribe Online.

Starting/Stopping Solutions

When you post a command to start or stop a Solution, if the Agent service associated with the Solution is stopped, the command waits up to 1 day for the service to be started before starting or stopping the Solution. Solutions that are in an invalid state for the command to execute are ignored.

For example:

  • A start command is ignored if the Solution is in an incomplete state or is already running.
  • A stop command is ignored if the Solution is not running.
